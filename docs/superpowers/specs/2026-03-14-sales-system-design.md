# Coach Unick 세일즈 시스템 설계

## 1. 개요

@coach_unick 인스타그램 기반 세일즈 시스템.
뷰퍼센트무브(ViewPercent Move) — 에이블리 공식 광고대행사 + 유료코칭.

**목표**: 월 매출 2,000만원 달성을 위한 퍼널 최적화 + AI 세일즈 코치 챗봇

## 2. 비즈니스 컨텍스트

### 매출 라인

| 상품 | 가격 구조 | 예상 객단가 (월) | 비고 |
|------|----------|-----------------|------|
| 광고대행 | 월 집행비의 10% | 200~500만원 | 광고비 2,000~5,000만원 집행 고객 기준 |
| 1회 코칭 | 30만원(+VAT), 화상미팅+속기록 | 33만원 | 리드 전환용 — 대행/구독으로 업셀 |
| 월 구독 코칭 | 월 100만원 + 매출증분 인센티브 | 100만원+ | 하이스트 매출 증분의 약속된 % |
| 프로젝트 단위 | 과업 규모별 | 100~300만원+ | 단발성, 더 높은 과업도 가능 |

> **참고**: @coach_unick 세일즈 시스템 전용 상품 라인. 기존 OS 코칭/MOVE 모듈과는 별도 라인.

### 현황

- 인스타 팔로워: 6,000명
- 월 DM 문의: 3~5건
- 응대 방식: 반구조화 (기본 멘트는 있으나 체계화 안 됨)
- 카카오톡 전환율: 30~50% 추정

### 월 매출 2,000만원 달성 시나리오

| 상품 | 건수 | 매출 |
|------|------|------|
| 상품 | 유지 고객 | 신규 (월) | 건당 매출 | 소계 |
|------|----------|----------|----------|------|
| 광고대행 3,000만원급 | 2건 | 2건 | 300만원 | 1,200만원 |
| 월 구독 코칭 | 2건 | 1건 | 100만원 | 300만원 |
| 프로젝트 | - | 2건 | 200만원 | 400만원 |
| 1회 코칭 (리드 전환용) | - | 3건 | 30만원 | 90만원 |
| **합계** | **4건** | **8건** | - | **1,990만원+α** |

> 유지 고객 4건은 누적. 월 신규 계약 목표는 약 8건이지만,
> 1회 코칭 3건은 리드 전환용이므로 **핵심 신규 계약 목표는 월 5건**.

## 3. 퍼널 설계: 3단계 압축 모델

7단계 퍼널 대신 전환이 일어나는 3개 병목에 집중:

```
콘텐츠 → DM 유입        [병목1: CTA 설계]
DM → 카카오톡 전환       [병목2: 30초 내 브릿지]
카카오톡 → 계약          [병목3: 진단→제안→클로징]
```

### 병목1: 콘텐츠 → DM (목표: 월 15건+)

- 모든 콘텐츠에 단일 키워드 CTA: "DM으로 '진단' 보내세요"
- 진입 장벽 최소화 — 긴 문의 대신 한 단어
- 주 5회 발행으로 도달 확대

### 병목2: DM → 카카오톡 (목표: 전환율 70%)

- DM 3회 이내에 카카오톡으로 이동
- DM에서는 진단/공감만, 상세 상담은 카카오톡
- 브릿지 멘트: "자세한 내용은 카카오톡에서 드릴게요" + 링크

### 병목3: 카카오톡 → 계약 (목표: 전환율 40%)

1. 현황 진단 질문 3개
2. 문제 공감 + 데이터 근거 제시
3. 맞춤 패키지 제안
4. 시작일 확정 (가격 협상이 아닌 일정 확정으로 프레이밍)

### 목표 역산

- 필요 핵심 신규 계약: 월 5건 (대행2 + 구독1 + 프로젝트2)
- 필요 1회 코칭 (리드 전환): 월 3건
- 필요 카카오톡 상담: 월 12~15건 (계약률 40% 기준, 코칭 포함)
- 필요 DM 문의: 월 18~22건 (카카오톡 전환률 70%)
- 필요 콘텐츠: 주 5건 발행

## 4. 콘텐츠 전략

### 주간 캘린더

| 요일 | 유형 | 목적 | CTA |
|------|------|------|-----|
| 월 | 캐러셀: 에이블리 광고 팁 | 전문성 입증 | DM "진단" |
| 화 | 릴스: 클라이언트 성과 B/A | 사회적 증거 | DM "진단" |
| 수 | 스레드: 업계 인사이트 | 도달 확장 | 프로필 유도 |
| 목 | 캐러셀: 대행사 선택 기준 | 코칭 니즈 자극 | DM "진단" |
| 금 | 스토리: 주간 성과 요약 | 직접 전환 | DM "진단" |

### 콘텐츠 유형별 설계

**캐러셀 (전환용 핵심 콘텐츠)**:
- 5~7장 구성
- 1장: 후킹 (질문 or 충격적 수치)
- 2~5장: 핵심 내용 (데이터 기반)
- 6장: 요약 + CTA
- 7장: "DM으로 '진단' 보내주세요"

**릴스 (도달 확장)**:
- 15~30초
- Before/After 성과 중심
- 자막 필수 (음소거 시청 대응)

**스레드 (브랜딩)**:
- 에이블리/지그재그 시장 인사이트
- 광고비 효율, ROAS 트렌드
- 대행사 선택 기준

## 5. DM 응대 시나리오

### 진입 분기

```
"진단" 키워드 수신
  └→ 자동 응답: "안녕하세요! 에이블리 광고 진단에 관심 가져주셔서 감사해요.
      혹시 현재 에이블리에서 광고 운영 중이신가요?"

응답 분기:
  ├→ "네, 운영 중이에요" → 대행 니즈 탐색 플로우
  ├→ "아니요, 아직이요" → 코칭 니즈 탐색 플로우
  └→ 기타 질문 → 상황 파악 후 분기
```

### 대행 니즈 탐색 플로우

```
Q1: "월 광고비 어느 정도 집행하고 계세요?"
  ├→ 1,000만원 이상 → 고가치 리드 (즉시 카카오톡 전환)
  ├→ 500~1,000만원 → 중가치 리드 (1회 코칭 제안)
  └→ 500만원 미만 → 저가치 (콘텐츠 팔로우 유도)

Q2 (고가치): "현재 직접 운영이세요, 대행사 사용 중이세요?"
  ├→ 직접 운영 → "카카오톡에서 무료 진단 해드릴게요" [브릿지]
  └→ 대행사 사용 중 → "어떤 부분이 가장 아쉬우세요?" → 불만 파악 후 [브릿지]
```

### 코칭 니즈 탐색 플로우

```
Q1: "어떤 부분이 가장 궁금하세요?"
  ├→ 광고 세팅/운영 → 1회 코칭 제안
  ├→ 대행사 평가/선택 → 1회 코칭 제안
  └→ 전반적 컨설팅 → 프로젝트 제안

[브릿지]: "자세한 내용은 카카오톡에서 말씀드릴게요.
여기로 연락 주시면 바로 상담 잡아드릴게요 → [카카오톡 링크]"
```

### DM 응대 원칙 (톤앤매너)

- 두괄식: 핵심 먼저, 배경은 뒤
- 부정어 연속 금지 ("안 되고", "못 하고" 연달아 X)
- "꼭/무조건" 남용 금지
- 3회 이내 카카오톡 전환 — DM에서 길게 끌지 않기
- 질문 1개씩 — 한 번에 여러 질문 금지

## 6. 카카오톡 상담 시나리오

### 상담 4단계

**1단계: 현황 진단 (5분)**
```
"반갑습니다! 먼저 현재 상황을 좀 여쭤볼게요."
Q1: "현재 에이블리 월 매출이 어느 정도 되세요?"
Q2: "광고비는 월 얼마 정도 집행하고 계세요?"
Q3: "현재 ROAS가 어떻게 나오고 있나요?"
```

**2단계: 문제 공감 + 데이터 제시 (5분)**
```
"[고객 상황] 상황이시군요. 저희가 비슷한 규모의 브랜드를 운영했을 때
ROAS [X]%에서 [Y]%로 개선된 사례가 있어요."

※ 반드시 구체적 수치 제시 — "잘할 수 있어요" 같은 모호한 표현 금지
```

**3단계: 맞춤 제안 (5분)**

고객 유형별 제안 로직:

| 월 광고비 | 추천 상품 | 제안 프레이밍 |
|----------|----------|-------------|
| 3,000만+ | 광고대행 | "현재 대비 ROAS X% 개선 목표로 운영" |
| 1,000~3,000만 | 대행 or 1회코칭→대행 전환 | "먼저 진단 후 방향 잡고 시작" |
| 1,000만 미만 | 1회 코칭 or 월구독 | "효율 먼저 올리고 스케일업" |
| 대행사 불만 | 코칭→대행 전환 | "기준부터 세우고 결정" |

**4단계: 클로징 (3분)**
```
"그러면 [날짜]부터 시작하는 걸로 진행할까요?"

※ 가격 협상이 아닌 일정 확정으로 프레이밍
※ "생각해볼게요" 대응: "그럼 [날짜]까지 결정해주시면
  [인센티브: 첫 달 세팅비 면제 등] 적용해드릴게요"
```

### 업셀 타이밍

- 대행 고객 → 계약 2개월 차에 코칭 제안 ("광고만으로는 한계, 브랜딩도 같이")
- 코칭 고객 → 코칭 완료 시점에 대행 제안 ("직접 실행은 저희가")
- 1회 코칭 → 무조건 다음 스텝 제안 (월구독 or 프로젝트 or 대행)

## 7. 세일즈 코치 챗봇 설계

### 핵심 기능

**입력**:
- 고객 대화 캡처 이미지 (스크린샷)
- 선택적 텍스트 메모 (추가 컨텍스트)

**AI 분석 출력 (한 화면)**:

1. **고객 진단**
   - 예산 규모 추정
   - 불만 유형 분류 (ROAS, 소통, 데이터, 기타)
   - 구매 의향 단계 (초기탐색 / 비교검토 / 결정직전)
   - 예상 객단가

2. **추천 응대 멘트 (3가지 톤)**
   - 부드러운: 공감 중심
   - 전문적: 데이터/논리 중심
   - 클로징: 행동 유도 중심

3. **다음 액션 추천**
   - 1순위/2순위 액션
   - 주의사항 (이 고객에게 하면 안 되는 것)
   - 업셀 타이밍 판단

### 시스템 프롬프트 구조

```
[세일즈 플레이북]
- 퍼널 3단계 모델
- DM 시나리오 전체
- 카카오톡 상담 4단계
- 업셀 타이밍 로직
- 톤앤매너 원칙

[고객 분석 프레임워크]
- 예산 규모 판단 기준
- 불만 유형 분류 기준
- 구매 단계 판단 기준

[응대 멘트 생성 규칙]
- 두괄식
- 부정어 연속 금지
- 개인화 필수
- 톤 자연스럽게 유지
```

전략 문서(docs/)가 곧 AI의 플레이북 → 문서 업데이트 = AI 성능 업데이트

```
│                  │                          │
│  [추가 메모 입력]  │  💬 추천 응대              │
│  "현재 타 대행사  │  [부드러운] [전문적] [클로징] │

├──────────────────┴──────────────────────────┤

```

### 대화 히스토리

- 같은 고객의 이전 대화 분석 결과를 참조할 수 있음
- localStorage에 저장 (**분석 결과 텍스트만** — 이미지는 저장하지 않음)
- 고객별 진행 상황 추적 (DM → 카카오톡 → 상담 → 계약)

**데이터 스키마**:
```json
{
  "id": "uuid",
  "customerName": "브랜드명 or 태그 (수동 입력)",
  "stage": "dm | kakaotalk | consulting | contract",
  "analyses": [{
    "timestamp": "ISO 8601",
    "memo": "추가 메모 텍스트",
    "diagnosis": { "budget": "", "painType": "", "buyingStage": "", "expectedValue": "" },
    "suggestedResponses": { "soft": "", "professional": "", "closing": "" },
    "nextActions": [""]
  }]
}
```
- 이미지는 분석 후 버림 (localStorage 용량 보호, 5MB 이하 유지)
- 고객당 최근 20건까지 보관, 초과 시 오래된 것부터 삭제

### 에러 처리

- API 호출 실패: "분석에 실패했습니다. 다시 시도해주세요" + 재시도 버튼
- 이미지 크기 초과 (10MB+): 업로드 전 리사이즈 (max 1568px)
- 대화 캡처가 아닌 이미지: AI가 "대화 내용을 찾을 수 없습니다" 응답 → 안내 메시지
- localStorage 만료: 가장 오래된 분석 자동 정리 + 경고

```
│   │   └── storage.js              # localStorage 관리
│   └── prompts/                    # (빌드 타임에 docs/sales-playbook.md를 import)

│   └── index.html
├── server.js                       # 간단 프록시 서버 (API 키 보호)

├── package.json

```

### 기술 스택

- **빌드**: Vite
- **UI**: React 18 + Tailwind CSS
- **AI**: Claude API (claude-sonnet-4-20250514) — 멀티모달 (이미지+텍스트)
- **저장**: localStorage (분석 결과 텍스트만, 이미지 미저장)
- **API 프록시**: Express 미니 서버 (API 키 서버측 보관, 브라우저 노출 방지)
- **모델**: 환경변수 `MODEL_ID`로 설정 가능 (기본값: claude-sonnet-4-20250514)
- **모바일**: 반응형 레이아웃 (모바일에서 1컬럼, 데스크톱에서 2컬럼)

### API 호출 구조

**프록시 서버** (`server.js`): Express로 `/api/analyze` 엔드포인트 제공.
API 키는 서버의 `.env.local`에만 존재, 브라우저에 노출되지 않음.

```
브라우저 → POST /api/analyze (이미지 base64 + 메모)
  → server.js → Anthropic API (API 키 서버측)
  → 분석 결과 JSON 반환
```

**플레이북 로딩**: `docs/sales-playbook.md`를 빌드 타임에 raw string으로 import.
`src/prompts/sales-coach.md`는 사용하지 않음 — `sales-playbook.md`가 유일한 소스.

```javascript
// server.js에서의 API 호출
{
  model: process.env.MODEL_ID || "claude-sonnet-4-20250514",
  max_tokens: 2000,
  system: salesPlaybookPrompt,  // docs/sales-playbook.md 내용
  messages: [{
    role: "user",
    content: [
      { type: "image", source: { type: "base64", media_type: "image/png", data: captureImage } },
      { type: "text", text: additionalContext }
    ]
  }]
}
```
